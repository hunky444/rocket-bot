# Telegram Gift Analyst Bot MVP

Стартовый каркас Telegram-бота для аналитики по подаркам в Telegram.

Сейчас в проекте есть:

- бот на `aiogram`
- базовые команды
- inline-меню внутри Telegram
- простая тарифная модель `Free / Pro / Premium`
- ограничение запросов для бесплатного плана
- локальное постоянное хранение пользователей и usage-лимитов через `SQLite`
- demo-paywall flow и demo-checkout под Telegram Stars
- market provider слой с market snapshots в БД
- готовая структура для дальнейшего подключения БД, MTProto и реального рынка

## Что умеет MVP

- `/start` — приветствие и меню
- `/help` — список возможностей
- `/plans` — тарифы
- `/top` — топ идей
- `/budget 5000` — что купить на указанный бюджет
- `/sell` — советы по продаже
- экран портфеля как premium teaser
- demo-портфель с сохранением позиций
- персональные budget-alerts
- обычные текстовые запросы вроде `что купить на 3000`
- свободные текстовые запросы как основной сценарий
- карточки подарков с фото, названием, моделью и фоном
- быстрые inline-кнопки внутри сообщений
- demo-покупка тарифа через checkout flow
- отображение источника и времени обновления данных

## Быстрый запуск

1. Установить Python 3.11+.
2. Создать виртуальное окружение.
3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Скопировать `.env.example` в `.env` и вставить токен бота.
5. Если хочешь подключать живые подарки Telegram, добавь `API_ID`, `API_HASH` и `SESSION_NAME`.
5. Запустить:

```bash
python -m app.main
```

## Первый вход Telethon

Если хочешь подключать реальные gifts и реальные изображения из Telegram, после заполнения `API_ID` и `API_HASH` выполни:

```bash
python -m app.collector.login_telethon
```

На первом запуске Telegram попросит:

- номер телефона
- код подтверждения из Telegram
- пароль 2FA, если он включен

После этого появится локальная session для MTProto.

## Синхронизация реальных подарков

После успешного логина Telethon можно загрузить реальные подарки Telegram в локальную базу:

```bash
python -m app.collector.sync_real_gifts
```

Этот шаг вызывает официальный MTProto-метод `payments.getStarGifts`, скачивает медиа подарков локально и заменяет содержимое таблицы `market_snapshots` живыми данными.

## Как устроен demo-flow

- новый пользователь попадает в `Free`
- основной сценарий общения идет через обычный текст, а кнопки остаются быстрыми shortcut-ами
- `Free` имеет лимит на budget-запросы
- в тарифах можно открыть demo-checkout для `Pro` или `Premium`
- после подтверждения оплаты создаются записи в `payments` и `subscriptions`
- состояние рынка берется из таблицы `market_snapshots`
- позиции портфеля хранятся в `portfolio_items`
- персональные алерты хранятся в `alerts`
- состояние хранится в файле `bot.db`

## Важно

Сейчас ответы строятся на данных из локальной таблицы `market_snapshots`, которая заполняется demo-значениями при первом запуске. Это промежуточный слой, чтобы быстро поднять бота, проверить UX и затем заменить источник на реальный collector.
